This is NOT the checkout — the checkout uses shopify/custom-pixel.js. What it does: 1. Loads posthog-js and inits the project. 2. Mirrors PostHog's distinct_id into a first-party cookie `fw_distinct_id` scoped to .fitwellbuckle.co so the sandboxed checkout pixel can read the same identity (the bridge the official PostHog Shopify guide omits). 3. Captures first-touch UTM/referrer once per session: - as PostHog person properties ($set_once), and - POSTed to https://admin.fitwellbuckle.co/api/tracking/utm, and - written to the Shopify cart as the `_fw_distinct_id` attribute so the order webhook has a deterministic server-side link too.
